DAU Ops Agent — 架構設計 v4

版本：v4.0  |  日期：2026-03-31  |  負責人：Jarvis

執行摘要

本文件定義 DAU Ops Agent 的完整建置架構，對齊 Claude 官方 Skill 標準（Agent Skills open standard），支援安裝、分享、跨 project 複用，並實現完全自動化 pipeline。

v3 → v4 三個核心調整
1. 整體從「project 資料夾」改為「可安裝 Skill」格式
2. 名詞對齊官方：Playbook → Workflow（WF），Skill → Module（M）
3. Frontmatter 加入 context: fork，實現 sub-agent 自動執行

一、名詞對齊表

v3 名稱 v4 名稱 官方對應 說明
Agent（整體） Skill Skill 整個 DAU Ops Agent 打包為一個可安裝 Skill
agent-context.md SKILL.md 前段 SKILL.md frontmatter + body 身份定義、邊界、路由邏輯，永遠載入
Playbook（PB-01～07） Workflow（WF-01～07） references/ 下的流程文件 任務觸發後按需 Read，不預載
Skill（SK-01～12） Module（M-01～12） references/ 下的知識文件 由 Workflow 指定 Read，按需載入
reference/ references/context/ references/ 靜態背景知識（歷史案例、基準值等）
（無） examples/ examples/ Few-shot 示範，官方明確支援

二、分層載入邏輯

官方 Skill 採用三層 Progressive Disclosure，與 v3 的 L1/L2/L3 設計完全吻合：

層級 載入時機 對應內容 Token 預算
L1 Metadata 永遠在 context SKILL.md frontmatter（name + description） ~100 tokens
L2 SKILL.md body Skill 觸發時載入 身份定義 + 任務路由邏輯 300–500 tokens
L3 References Claude Read 時才進 context Workflow 文件 + Module 文件 + 背景知識 400–1,000 tokens / 件
Token 效率：一次完整任務 ≈ L1（100）+ L2（400）+ 1 個 WF（400）+ 2–3 個 M（900）= ~1,800 tokens。其餘 6 個 WF 和 9 個 M 完全不在 context window。

四、SKILL.md 設計規範

Frontmatter（決定觸發與執行模式）

---
name: dau-ops-agent
description: >
  DAU Ops 活躍營運 AI Agent，專責 CODM 活躍組全部高頻工作流。
  當用戶提到以下任何情境時自動觸發：活動設計、活動規劃、DAU 異動、
  指標下降、留存率、週報、上線審查、QA、活動復盤、競品分析、
  玩家流失、回流策略、推播設計。
context: fork
effort: high
---
欄位 設定值 原因
context: fork fork 每次任務在獨立 sub-agent 執行，不汙染主對話；實現無人介入 pipeline
effort: high high 確保複雜任務有足夠推理深度
disable-model-invocation （不設定，保持預設 false） 讓 Claude 根據 description 自動判斷觸發，不需要手動 /dau-ops-agent

SKILL.md Body 結構

Body 只做三件事，保持精簡：

  1. 身份定義：我是誰、負責什麼、邊界在哪（取代原 agent-context.md）
  2. 任務路由表：識別 7 種情境 → 指定 Read 對應 WF 文件
  3. 通用輸出規則：語言、格式、完成標準

七、新增功能：動態 Context 注入

官方支援 !`command` 語法，在 Skill 觸發前執行 shell command 並將結果注入 context。這是 WF-03 週報自動化的關鍵，無需人工貼數據：

## 本次任務 Context（自動注入）

- 本週指標快照：!`python scripts/fetch_metrics.py`
- 本週活動紀錄：!`cat references/context/event-log.md`
- 當前日期：!`date +%Y-%m-%d`

Skill 觸發時，以上 command 先執行，輸出結果替換進 prompt，Claude 收到的是真實數據。

八、觸發控制對照

設定 你能觸發 Claude 自動觸發 適用情境
（預設，不設任何欄位） ✓ ✓ DAU Ops Agent 採用此設定，全自動
disable-model-invocation: true ✓ ✗ 有副作用的操作（如 deploy、發推播），需手動觸發
user-invocable: false ✗ ✓ 純背景知識，不應暴露為指令

九、建議建置順序

階段 工作項目 說明
Phase 1 SKILL.md 身份定義 + 路由邏輯，先跑通觸發機制
Phase 2 M-02、M-03 複用率最高的兩個 Module，先建
Phase 3 WF-01 + 對應 M 活動設計是最高頻任務，做第一個端到端測試
Phase 4 WF-02、WF-03 指標異動分析 + 週報，加入動態 context 注入
Phase 5 剩餘 WF + M 補齊 WF-04～07 與對應 Module
Phase 6 Evals + 打包 用 skill-creator 跑測試，打包成 .skill 分發

十、分發與安裝

完成後使用官方打包指令：

python -m scripts.package_skill dau-ops-agent/

產出 dau-ops-agent.skill 檔案，團隊成員直接安裝即用，不需要複製資料夾或手動設定。

安裝層級 路徑 適用對象
個人 ~/.claude/skills/dau-ops-agent/ Jarvis 本人，所有 project 可用
Project .claude/skills/dau-ops-agent/ 特定 project 的成員
Plugin <plugin>/skills/dau-ops-agent/ 透過 Plugin 分發給整個團隊
