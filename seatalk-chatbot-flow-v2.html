03 — Signing Secret 驗簽機制

驗簽步驟
  1. 取 callback request body（原始 bytes）
  2. 串接 Signing Secret（字串形式）
  3. 計算 SHA-256 hash
  4. Hex encode（全小寫）
  5. 比對 HTTP header 的 Signature 欄位
Python 範例
import hashlib def is_valid(secret, body, sig): return hashlib.sha256( body + secret ).hexdigest() == sig

04 — Event Types 清單

Event Type 觸發情境 你的 Agent 需要處理？
message_from_bot_subscriber 使用者在 DM 傳訊給 Bot ✅ 核心事件，必須處理
new_mentioned_message_from_group_chat Group Chat 中 @Bot ✅ 核心事件，必須處理
interactive_message_click 使用者點擊 Interactive Message 按鈕 ✅ 若有互動卡片功能需處理
new_message_received_from_thread Bot 被提及的 Thread 有新回覆 視需求，可接入 Thread 對話
user_enter_chatroom_with_bot 使用者進入與 Bot 的對話室 可用於歡迎訊息
bot_added_to_group_chat Bot 被加入群組 可用於自我介紹訊息
bot_removed_from_group_chat Bot 被移出群組 Log 用途
event_verification Callback URL 初次設定驗證 ✅ 必須處理（一次性）

05 — Send Message Payload 格式

DM（1-on-1）文字訊息
{ "employee_code": "EMP123", "message": { "tag": "text", "text": { "content": "你的回覆內容", "format": 1 } } }
Interactive Message（互動卡片）
{ "employee_code": "EMP123", "message": { "tag": "interactive_message", "interactive_message": { "elements": [ { "tag": "header", "text": {...} }, { "tag": "section", "text": {...} }, { "tag": "action", "elements": [ { "tag": "button", "text": {"content": "確認"}, "value": "confirm" } ] } ] } } }
注意：Send Message 用的識別碼是 employee_code，不是 uid。如果你只拿到 email，先呼叫 get_employee_code_with_email 換取 employee_code 再送訊息。

06 — 串接 Checklist

  • SeaTalk SOP Portal 建立 Bot App，取得 app_id / app_secret / Signing Secret
  • Agent Server 建立 /callback：處理 event_verification（回傳 seatalk_challenge），完成 URL 驗證
  • 實作 SHA-256 驗簽（每次 callback 都要驗）
  • 實作 access_token 取得 + 自動刷新（TTL 7200 秒）
  • 處理 message_from_bot_subscriber（DM）和 new_mentioned_message_from_group_chat（Group）兩種核心事件
  • Callback endpoint 必須 5 秒內回 200；長任務改 async，完成後再主動送訊息
  • 決定串接路徑：自行維護 API 邏輯（Path A）或安裝 seatalk-mcp-server 讓 Claude 直接用（Path B）
  • 若用 Interactive Message，處理 interactive_message_click 的 callback
  • End-to-end 測試：DM → callback → Claude Agent → Bot 回覆
